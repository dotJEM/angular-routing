$route
service in module dotjem.routing

Description

Is used for deep-linking URLs to states. It watches $location.url() and tries to map the path to an existing route definition.

You can define routes through $routeProvider's API.

Dependencies

  • $location
  • $routeParams

Methods

  • change(args)

    Changes the route.

    As a result of that, changes the path.

    Parameters
    ParamTypeDetails
    argsObject

    Object with details about the route change. The route definition contains:

    • route {string} The route to change to, can have parameters.
    • params {Object=} A parameter object with parameters to fill into the route.
    • replace {boolean=} - True if the route should replace the browser history entry, otherwise false.

  • format(route, params, base)

    Formats the given provided route into an url.

    Parameters
    ParamTypeDetails
    routestring

    Route to format.

    params
    (optional)
    		Object

    Parameters to fill into the route.

    base
    (optional)
    		Boolean

    If false ignore the base path (default is true)

    Returns
    string

    An url generated from the provided parameters.

  • reload()

    Causes $route service to reload the current route even if $location hasn't changed.

    As a result of that, jemView creates new scope, reinstantiates the controller.

Properties

  • current

    Reference to the current route definition.

  • routes

    Array of all configured routes.
