States is the core concept and can be used to manage most aspects of an application build with dotJEM Angular Routing.

Routes

Through the route property, states can be bound to a route which can then be used to activate the state.

Routes are similar to those of ngRoute but with a number of enhancements like parameter converters.

Hierarchies

Configuring states in a hierarchy gives more control over your application with less code.

This fits perfectly into the web where users drill down to find the information they need.

Links

Using the sref directive, it is possible to link directly to states within your application.

This ensures that if routes bound to states changes, there is no need to change links throughout the application.

The sample goes beyond ngRoute when we look at the markup however.

When we look at the menu we will notice the use of <a sref="'about'">...</a> which tells the application that this link should target the state about so that if we changed the route of that state to /newabout in the configuration, the link would automatically be rewritten as well.

We also use an isActive function bound the $state services isActive function to tell us when a given state is active to highlight a menu item.

State Transitions

$state.goto('B/C');
In this example we transition from the root state R to B.C which causes the states B and B.C to be activated causing their views to be loaded. 

$state.goto('../C');
In this example we transition between two siblings, B.A and B.C under B, this will deactivate B.A and activate B.C. The state B will be left intact.
