Views

Views are used render the user interface, views can be parallel as well as nested in each other and an application can define as many views as needed.

Templates

A single property, template that automatically detects if it's a url, function or a simple html string.

Alternatively a template object can be provided that defines either url, fn or html to explicitly define sort of template it is.

Controllers

Defined on each view if needed, controllers will be bound to the view element.

Just like with the ngRoute module, controllers are used to provide the data to the view.

View Service

Views can be managed through the $view service, the same service as the state engine uses.

Views can be created, refreshed, updated and removed.

A transactional concept is supported for grouping multiple changes into a single commit.

template

The sample demonstrates three ways of declaring templates:

url:
If the content parses as an URL it will be treated as such.
function:
Functions can be passed that returns the template, array annotated functions can be used.
html:
If the content does not parse as an URL it will be considered to be the actual template.

Templates can also be defined by passing an object with either a url, fn or html property, this is useful when 'templates/home.html' was actually meant to be treated as a html string rather than a URL.
