jemView (directive in module dotjem.routing )

Description

Overview

jemView is a directive that complements the $state service by including the rendered template of the current state into the main layout (index.html) file. Every time the current route changes, the included view changes with it according to the configuration of the $state service.

animations

  • enter - animation is used to bring new content into the browser.
  • leave - animation is used to animate existing content away.

The enter and leave animation occur concurrently.

Usage

as element:
<jem-view
       name="{string}">
</jem-view>
as attribute
<ANY jem-view="{string}">
   ...
</ANY>
as class
<ANY class="jem-view: {string};">
   ...
</ANY>

Parameters

  • jemView|name – {string} –

    Name of the view

Events

  • $refresh

    This event is broadcasted on the view scope unless the view scope defines a refresh function.
    Refresh happens for sticky views when the sticky flag remains the same during an update.

    Type:

    broadcast

    Target:

    the current jemView scope

    Parameters

    • angularEvent – {Object} –

      Synthetic event object.

    • name – {string} –

      The name of the view where the broadcast originated.

    • name – {Object} –

      Any data that may have been provided for a refresh.

  • $viewContentLoaded

    Emitted every time the jemView content is reloaded.

    Type:

    emit

    Target:

    the current jemView scope
