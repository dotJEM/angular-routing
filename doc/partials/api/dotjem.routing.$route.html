$route (service in module dotjem.routing )

Description

Is used for deep-linking URLs to states. It watches $location.url() and tries to map the path to an existing route definition.

You can define routes through $routeProvider's API.

Dependencies

  • $location
  • $routeParams

Methods

  • change(args)

    Changes the route.

    As a result of that, changes the path.

    Parameters

    • args – {Object} –

      Object with details about the route change. The route definition contains:

      • route {string} The route to change to, can have parameters.
      • params {Object=} A parameter object with parameters to fill into the route.
      • replace {boolean=} - True if the route should replace the browser history entry, otherwise false.

  • format(route, params)

    Formats the given provided route into an url.

    Parameters

    • route – {string} –

      Route to format.

    • params(optional) – {Object=} –

      Parameters to fill into the route.

    Returns

    {string} –

    An url generated from the provided parameters.

  • reload()

    Causes $route service to reload the current route even if $location hasn't changed.

    As a result of that, jemView creates new scope, reinstantiates the controller.

Properties

  • current

    Reference to the current route definition.

  • routes

    Array of all configured routes.

Events

  • $routeChangeError

    Broadcasted if any of the resolve promises are rejected.

    Type:

    broadcast

    Target:

    root scope

    Parameters

    • angularEvent – {Object} –

      Synthetic event object.

    • current – {Route} –

      Current route information.

    • previous – {Route} –

      Previous route information.

    • rejection – {Object} –

      Rejection of the promise. Usually the error of the failed promise.

  • $routeChangeStart

    Broadcasted before a route change. At this point the route services starts resolving all of the dependencies needed for the route change to occurs.

    Type:

    broadcast

    Target:

    root scope

    Parameters

    • angularEvent – {Object} –

      Synthetic event object.

    • next – {Route} –

      Future route information.

    • current – {Route} –

      Current route information.

  • $routeChangeSuccess

    Broadcasted after a route dependencies are resolved.

    Type:

    broadcast

    Target:

    root scope

    Parameters

    • angularEvent – {Object} –

      Synthetic event object.

    • current – {Route} –

      Current route information.

    • previous – {Route|Undefined} –

      Previous route information, or undefined if current is first route entered.

  • $routeUpdate

    The reloadOnSearch property has been set to false.

    Type:

    broadcast

    Target:

    root scope
