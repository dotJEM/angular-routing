Adds a new route definition to the $route service.

Parameters:

  • path String

    Route path (matched against $location.path). If $location.path contains redundant trailing slash or is missing one, the route will still match.

    path can contain named groups starting with a colon (:name) or curly brackets ({name}). All characters up to the next slash are matched and stored in $routeParams under the given name when the route matches.

    Further, when using the curly bracket syntax, converters can be used to match only specific values, ({num:name}) will only match numerical values and ({regex(\d{1,2}[a-z]+):name}) would only match a parameter starting with one or two digits followed by a number of characters between 'a' and 'z'.

    More converters can be registered using the {@link ui.routing.routeProvider#convert convert} function.

  • route Object

    Mapping information to be assigned to $route.current on route match.

    Object properties:
