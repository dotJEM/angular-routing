Routes

Routes is primarily used to connect states to urls, the route API is almost identical to the ngRoute api with a few additional features.

Parameter Converters

Through parameter converters, routes can be much more specific.

E.g. if only numeric id's should be allowed the num converter can be used to only let numeric values pass.

State Activators

While states can be activated through routes by using the route property.

If multiple routes should lead to the same state a route can simply define the state property as well.

Decorators

Decorators can be registered to alter routes before they update events are broad casted.

This allows for altering the outgoing route objects.

It also configures a parameter converter than is used to only accept parameters within a given numeric range.

Finally it defaults to the route /404 if none of the defined routes are matched.

home.html

In the home.html template we have added a number of extra links to the page.

Since we are activating states though their alternative routes we can't use the sref directive here since that would obviously pick the primary route of each state. So instead the links are hardcoded.

There are two valid links as well as two invalid ones. Clicking the valid ones will correctly bring us to the desired states, however if we click the invalid ones we go to the 404 page.

Scenarios

Configuring alternative routes can be used to provide short-hand routes or keep old routes that has been replaced in an application active.

In both cases a redirection might be more suited however, which is achieved though the same API.
